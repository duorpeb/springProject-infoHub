> # 💡 프로젝트 소감
 
### **1. API 의 사용**
- 이전 팀 프로젝트를 하며 API 를 한 번 사용해봤을 때, 느낀 점은 API 를 사용하기 전 API 가 어떤 형식으로 데이터를 뿌려주는지 알면 좋을 것 같다는 생각이 들었었습니다. 그래서 이런 저런 Tool 을 찾아보다가 Postman 을 이용해 네이버 검색 API 와 기상청 단기 예보 API 가 어떤 형식으로 데이터를 뿌려주는지 미리 파악하고 API 를 사용했고 이는 설계 단계에서 페이지의 구조를 설계할 때, 이전보다 디테일한 설계를 할 수 있게 해주었던 것 같습니다. <br><br>

- 처음엔 기상청 단기 예보 API 와 기상청 중기 예보 API 를 활용해 현재 사용자 위치의 1주일 간의 날씨 정보를 출력하고 싶었지만 시간 상 기상청 단기 예보 API 만 활용해 4일간의 날씨 정보를 출력하게 되어 아쉬웠습니다.  

<br><br><br>


### **2. 더 나은 UI/UX 에 대한 고민**

- 설계 단계에서 네이버 검색 API 를 사용해 메인 페이지의 10개의 뉴스 기사를 출력할 때, 비동기 방식과 동기 방식 중 어느 방식이 사용자 경험을 향상시키는 방식일지 고민하다가 동기 방식으로 처리하는 것을 선택했는데 이유는 다음과 같습니다.

  1. 날씨 정보는 비동기로 처리 했기에 날씨 정보를 제외한 메인 페이지의 무게가 그리 무겂지 않다고 생각했습니다.
 
  2. 동기로 처리할 수 있는 것은 동기로 처리하는 것이 좋다고 생각했습니다. <br><br>
     

- 하지만 구현을 마친 뒤 수정과 보완 과정을 거치며 트래픽 급증 (스파이크) 에 능동적으로 대응하기 위해서는 동기 방식보다 비동기 방식이 더 적합하다는 사실을 알게 되었습니다. 커뮤니티 사이트를 개발했기에 트래픽에 대한 부분도 신경을 썼었어야 하는데 로컬 호스트 환경과 시간에 쫓겨 개발을 진행하다 보니 트래픽 과부하에 대한 고려는 미처 하지 못했고 지금와서 되돌아보니 이 부분을 신경쓰지 못한 것이 제 실수인 것 처럼 느껴졌습니다.
